# MiroFish プロジェクト概要

**MiroFish** は、多エージェント（Multi-Agent）技術を活用した次世代のAI予測エンジンです。現実世界のニュース、政策、金融信号などの「種（シード）」から、独立した人格と記憶を持つ数千のエージェントが活動するデジタル空間を構築し、未来の社会演化をシミュレーションします。

## 核心的なコンセプト
- **「神の視点」による推演**: 変数を注入し、未来がどのように変化するかをデジタル砂盤（サンドボックス）上でシミュレート。
- **群体知能のミラーリング**: 個々のエージェントの相互作用から生じる「創発（Emergence）」を捉え、従来の予測手法の限界を突破。

## 技術スタック
- **Backend**: Python (Flask), `uv` (パッケージマネージャー)
- **Frontend**: Vue.js (Vite), TailwindCSS
- **Simulation Engine**: [OASIS](https://github.com/camel-ai/oasis) (by CAMEL-AI)
- **Knowledge Graph / Memory**: [Zep Cloud](https://www.getzep.com/) (長期記憶、エンティティ抽出)
- **LLM**: OpenAI互換API (Qwen, GPT-4o-mini等)

## 主要なワークフロー

### 1. グラフ構築 (Graph Construction)
アップロードされた文書（PDF, MD, TXT等）から、LLMがオントロジー（エンティティと関係性）を定義し、Zepを用いてナレッジグラフ（GraphRAG）を構築します。

### 2. シミュレーション準備 (Preparation)
グラフから抽出された実体に基づき、各エージェントのプロファイル（人格、記憶、行動論理）とシミュレーション設定（期間、頻度、トピック）を自動生成します。

### 3. シミュレーション実行 (Simulation)
TwitterやRedditを模した並行プラットフォーム上でエージェントを自由に相互作用させます。エージェントのアクティビティはリアルタイムでモニタリング可能です。

### 4. レポート生成と対話 (Report & Interaction)
シミュレーション結果をAI（ReportAgent）が分析し、詳細な予測レポートを出力します。また、シミュレーション世界のエージェントと直接対話して深掘りすることも可能です。

## プロジェクト構造
- `/backend`: FlaskベースのAPIサーバー。シミュレーションランナーやタスク管理を含む。
- `/frontend`: Vue.jsによるダッシュボード。グラフ可視化やリアルタイムログ表示。
- `/static`: ロゴやスクリーンショットなどのアセット。
- `docker-compose.yml`: コンテナ化されたデプロイ環境。
